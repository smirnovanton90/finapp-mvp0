# Решение проблем с запуском приложения

## Кнопка "Запустить" не активна

Если кнопка запуска приложения не активна, проверьте следующее:

### 1. Создание конфигурации запуска

1. В Android Studio выберите `Run` → `Edit Configurations...` (или нажмите на выпадающий список конфигураций рядом с кнопкой запуска)
2. Нажмите `+` (плюс) в левом верхнем углу
3. Выберите `Android App`
4. Заполните поля:
   - **Name**: `app` (или любое другое имя)
   - **Module**: выберите `app` из списка
   - **Launch**: выберите `Default Activity`
   - **Target**: выберите `USB Device` или `Emulator` (или оставьте пустым, чтобы выбрать при запуске)
5. Нажмите `OK`

### 2. Проверка синхронизации Gradle

1. Убедитесь, что проект синхронизирован:
   - Внизу экрана должна быть надпись "Gradle sync finished"
   - Если есть ошибки синхронизации, они будут показаны в панели "Build"
2. Если синхронизация не прошла:
   - Нажмите `File` → `Sync Project with Gradle Files`
   - Или нажмите на иконку слоненка в тулбаре (Sync Project with Gradle Files)

### 3. Проверка выбора устройства/эмулятора

1. В верхней панели инструментов должно быть поле выбора устройства (рядом с кнопкой запуска)
2. Убедитесь, что выбрано устройство:
   - **Физическое устройство**: подключите устройство через USB и включите режим отладки по USB
   - **Эмулятор**: создайте и запустите AVD (Android Virtual Device) через `Tools` → `Device Manager`

### 4. Проверка компиляции

1. Попробуйте собрать проект: `Build` → `Make Project` (или `Ctrl+F9`)
2. Если есть ошибки компиляции, они будут показаны в панели "Build"
3. Исправьте все ошибки перед запуском

### 5. Проверка модуля приложения

1. Убедитесь, что модуль `app` существует:
   - В панели Project (слева) должна быть папка `app`
   - В `settings.gradle.kts` должен быть `include(":app")`
2. Если модуль отсутствует:
   - Проверьте `settings.gradle.kts`
   - Выполните синхронизацию Gradle

### 6. Очистка и пересборка проекта

Если ничего не помогает:

1. `Build` → `Clean Project`
2. Дождитесь завершения
3. `Build` → `Rebuild Project`
4. Дождитесь завершения
5. `File` → `Invalidate Caches...` → `Invalidate and Restart`

### Быстрая проверка

1. ✅ Проект синхронизирован с Gradle (нет ошибок синхронизации)
2. ✅ Создана конфигурация запуска (`Run` → `Edit Configurations...`)
3. ✅ Выбрано устройство или эмулятор (в верхней панели инструментов)
4. ✅ Нет ошибок компиляции (`Build` → `Make Project`)
5. ✅ Модуль `app` виден в проекте

## Создание эмулятора (если нет устройства)

1. `Tools` → `Device Manager`
2. Нажмите `Create Device`
3. Выберите устройство (например, Pixel 7)
4. Выберите системный образ (например, Android 14, API 34)
5. Скачайте образ, если нужно
6. Завершите создание эмулятора
7. Нажмите ▶️ (Play) рядом с созданным эмулятором, чтобы запустить его
